Git Ajout de nouveaux fichiers

Vous venez de créer votre premier dépôt Git local. Mais il est vide.

Ajoutons donc quelques fichiers, ou créons un nouveau fichier à l’aide de votre éditeur de texte préféré. Ensuite, enregistrez-le ou déplacez-le dans le dossier que vous venez de créer.

Si vous souhaitez apprendre à créer un nouveau fichier à l’aide d’un éditeur de texte, vous pouvez consulter notre tutoriel HTML :
Éditeurs HTML

Pour cet exemple, je vais utiliser un simple fichier HTML comme celui-ci :

Exemple

<!DOCTYPE html>
<html>
<head>
<title>Hello World!</title>
</head>
<body>

<h1>Hello world!</h1>
<p>This is the first file in my new Git Repo.</p>

</body>
</html>

Et enregistrez-le dans notre nouveau dossier sous le nom de .index.html

Revenons au terminal et listons les fichiers dans notre répertoire de travail actuel :

Exemple

ls
index.html

ls listera les fichiers dans dans l’annuaire. Nous pouvons voir que c’est là.index.html

Ensuite, nous vérifions le Git et voyons s’il fait partie de notre dépôt :status

Exemple

git status
On branch master

No commits yet

Untracked files:
  (use "git add ..." to include in what will be committed)
    index.html

nothing added to commit but untracked files present (use "git add" to track)

Maintenant, Git est au courant du fichier, mais ne l’a pas ajouté à notre dépôt !

Les fichiers de votre dossier de dépôt Git peuvent être dans l’un des 2 états suivants :

  • Suivi : fichiers dont Git a connaissance et qui sont ajoutés au référentiel
  • Non suivi : fichiers qui se trouvent dans votre répertoire de travail, mais qui ne sont pas ajoutés au répertoire dépôt

Lorsque vous ajoutez pour la première fois des fichiers à un référentiel vide, ils sont tous non suivi. Pour que Git les suive, vous devez les mettre en scène ou les ajouter à la balise l’environnement de mise en scène.

Nous aborderons l’environnement de mise en scène dans le chapitre suivant.
